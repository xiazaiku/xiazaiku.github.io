CopyQ - 强大剪贴板增强工具 v11.0.0 官方便携版

CopyQ 是一款开源且使用 QT 框架编写的剪切板管理利器，为用户提供卓越的剪贴板管理体验。它不仅支持搜索、编辑历史等基础功能，更具备一系列高级特性，例如支持存储和处理包括图片在内的多种数据格式，以及灵活的命令行操作和自定义快捷键等。CopyQ 适用于 Linux、Windows 和 OS X 10.9+ 等主流操作系统。

核心功能特性：

  • 存储文本、HTML、图像以及其他任何自定义格式的数据。
  • 快速浏览和过滤剪贴板历史记录中的条目，迅速定位所需内容。
  • 通过选项卡对剪贴板项目进行排序、创建、编辑、删除、复制/粘贴、拖放等操作，管理更加井井有条。
  • 为每个剪贴板项目添加注释或标签，方便分类和查找。
  • 支持全局快捷键，并允许用户自定义命令，操作效率倍增。
  • 通过快捷方式或从系统托盘或主窗口直接粘贴项目，灵活便捷。
  • 提供完全可定制的外观，满足个性化需求。
  • 配备高级命令行界面和脚本编写功能，扩展性强大。
  • 可设置忽略从特定窗口复制的剪贴板内容，或包含特定文本的剪贴板，避免干扰。
  • 内置类似 Vim 的简易编辑器和快捷键绑定，方便文本编辑。

更新日志

v11.0.0 版本更新亮点：

  • 新增对深色/浅色窗口标题方案的支持，界面视觉效果更佳。
  • 引入 frameless_window 选项（#2570），可切换主窗口框架和标题栏（需窗口管理器支持）。
  • 在命令 INI 文件中增加对本地化命令名称的支持（#3032），支持多语言环境。
  • 增强图像格式预览支持，现在可显示 ico 和 webp 格式的预览。
  • 脚本中增加对更复杂的网络请求的支持，新的 NetworkRequest 类允许设置自定义标头、HTTP 方法、允许的重定向次数和超时时间。
  • 优化了避免隐藏退格窗口的机制（#3107）。
  • 启用菜单中的 Vi/Emacs 导航（#3012），并使 Vi 中的 Ctrl+[ 和 Emacs 中的 Ctrl+G 在更多场景下充当 Esc 键的功能。
  • 脚本中的选择和当前项目/行/数据现在仅与脚本中使用 tab（选择 tab(...) 的选项卡相关。
  • 从脚本执行命令和操作时，移除不必要的超时，提高执行效率。
  • 优化了 action()/execute() 函数，避免在命令不包含 %1 占位符时获取和传递剪贴板数据，进一步提升性能。
